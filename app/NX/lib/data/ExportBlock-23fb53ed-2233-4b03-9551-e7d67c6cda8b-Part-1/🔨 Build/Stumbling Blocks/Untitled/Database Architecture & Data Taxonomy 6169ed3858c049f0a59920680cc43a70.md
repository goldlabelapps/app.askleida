# Database Architecture & Data Taxonomy

Existential Risk: A poorly architected database — fragmented records, inconsistent taxonomy, unextractable data — costs millions at exit because the data asset cannot be cleanly verified or licensed. This is the single biggest technical risk to long-term valuation.
Mitigation Required: Three-database architecture from day one — Master Product Database, Practitioner Personal Database (row level security), Anonymised Proprietary Dataset (backend only, accumulates automatically). Rigorous normalised schema linking practitioner credentials, client concern taxonomy, recommended products and outcomes. Chris to architect this before a single real user record is created. No shortcuts.
